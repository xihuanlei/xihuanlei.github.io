如何把本地项目上传到gitee仓库
发表于2021-07-21|更新于2021-08-04
|字数总计:863|阅读时长:2分钟|阅读量:

准备工作

  • 创建一个仓库
  • 新建文件夹把本地项目拉入
  • 通过git命令上传项目到指定仓库

工具：👉git官网、👉gitee直达

正文开始

  1. 创建仓库

注意两点在设置模板勾选readme文件，选择分支模型选择master分支

  1. 创建一个文件夹，把要上传的项目复制进来

建议： 新建文件夹名和上传项目名都命名为英文

声明：这里我新建的文件夹为springbootz，下面我将用springbootz文件夹来做介绍

  1. 通过git命令上传项目

上传之前先看看需要那些命令以及这些命令的含义是什么

目标仓库=https://gitee.com/你的码云用户名/你创建的仓库名.git
注：在此命令运行期间，可能会要求输入你码云的用户名和密码，正确输入即可，一般是第一次使用会出现验证以及一些项目错误需要重新用户验证

① 把项目复制到新建的文件夹后，右击鼠标选择Git Bash Here，执行命令git init回车之后，springbootz文件夹会生成.git文件

② 连接远程目标仓库git remote add origin+目标仓库 + 代表空格

③ 拉取仓库文件到本地，git pull origin master

登录验证一下，执行命令 git push --set-upstream origin master

④ 将文件保存到缓存区，执行命令git add + 文件名，如果是一个文本文件它是由后缀名.txt的，这个时候我们需要执行命令git add + 文件名.txt

这里执行命令：git add internship

⑤ 描述文件信息，执行命令git commit -m '描述'
注意：单引号是英文状态下的单引号
这里执行命令：git commit -m ‘测试’

⑥ 将本地项目“internship”push到码云仓库，执行命令git push origin master
执行完命令，刷新页面就可以看到上传的项目文件出现在目标仓库了

上述步骤完成后，以后再上传项目只要执行以下几步就行了，前提是把要上传的文件放到spingbootz（新建）文件夹里

⑦ 删除文件
如果不小心把文件上传错了，可以找到对应文件，鼠标右键，一键删除。
